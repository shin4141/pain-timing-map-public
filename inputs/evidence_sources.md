# Evidence Sources

This file is a manual evidence log. It does not define scraping, automation, crawling, API collection, or generic search behavior.

## Source Log Template

```text
Source ID:
Date observed:
Source type:
Source title or description:
Source URL or local reference:
Observed audience:
Observed pain:
Evidence excerpt or summary:
Current workaround:
Signal notes:
Reliability notes:
```

## Source Types

Use plain source categories such as:

- Manual interview note
- Public discussion
- Product review
- Support thread
- Changelog or release note
- Forum post
- Social post
- Customer-facing documentation
- Personal observation

## Evidence Rules

- Keep evidence attached to a specific observed pain.
- Separate raw pain from interpretation.
- Mark uncertainty clearly.
- Do not treat volume alone as a build signal.
- Do not skip LLM absorption risk, user fit, or avoidance fit.
- Do not convert evidence into a build command.
