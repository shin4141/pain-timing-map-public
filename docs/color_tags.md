# Color Tags

Color tags are review-state markers for timing and evidence. They help a reader see how a pain signal is currently being treated during manual review.

Color tags are not medical labels. They are not recommendations. They do not decide what to build, buy, publish, or act on.

## Tag Meanings

- GREEN: signal is weak, resolved, or not currently actionable.
- YELLOW: signal needs recheck or more evidence.
- ORANGE: signal is meaningful, but timing or action remains uncertain.
- RED: signal is high-risk, urgent, or should not be acted on without review.
- BLUE: external or contextual update signal.
- GRAY: archived, stale, or currently not enough evidence.

## How To Use Them

Use one color tag as a quick visible state on a Pain Timing Map record. Pair it with evidence notes, timing notes, and a gate such as WATCH, HOLD, CAP, PREPARE, BUILD CANDIDATE, or reject.

Do not use a color tag by itself as a decision. If the evidence changes, the tag can change during a later manual recheck.

This is not a recommendation to build.
This is an evidence-backed pain signal map.
The build decision remains with the user.
