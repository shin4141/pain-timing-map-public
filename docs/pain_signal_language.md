# Pain Signal Language

## Core Idea

In Pain Timing Map, "pain" does not only mean discomfort.

A pain signal can be a sign of:

- missed opportunity
- wasted work
- rework
- upcoming failure
- decision error
- route mismatch
- timing mismatch

Pain is not automatically a stop signal.

Pain is first treated as evidence.

## Pain Is A Signal, Not A Verdict

A pain signal does not mean:

- stop immediately,
- abandon the project,
- build a product,
- sell something,
- automate the workflow,
- turn the signal into a task.

It means:

> Something may be changing, leaking value, wasting effort, creating rework, or moving toward failure.

The signal should be classified before action.

## Signal Types

| Signal type | What it means | Example |
|---|---|---|
| Opportunity signal | A chance may be missed if ignored too long | Users repeatedly describe a problem before a market category exists |
| Waste signal | Work may be consuming effort without improving the outcome | Repeated cleanup that does not change user value |
| Rework signal | A current path may create future correction cost | AI-generated work that later needs manual reconstruction |
| Risk signal | A small issue may become a larger failure | Stale handoff, unclear status, or missing next action |
| Stop signal | Continuing may create avoidable damage | Public release with private context still exposed |
| Route-adjustment signal | The goal may remain valid, but the path needs changing | Publish later, but first create a safer public surface |
| Observation signal | The signal is not actionable yet, but should not be discarded | A recurring small curiosity or weak market signal |

## Relation To Entry

Pain Timing Map provides signals.

Entry checks whether it is safe and useful to act now.

The flow is:

```text
Pain Signal
↓
Entry Window
↓
GO / SOFT GO / HOLD / BLOCK
↓
Smallest Next Action
```

Pain asks:

> What signal is appearing?

Entry asks:

> Is this the right time and route to act?

## Why This Matters

If every pain is treated as a stop signal, useful opportunities are lost.

If every pain is treated as a build signal, work explodes.

Pain Timing Map separates the signal from the action.

The signal is captured first.
The timing and gate are decided later.

## Boundary

This is not medical advice.
This is not financial advice.
This is not a recommendation to build, buy, sell, publish, or automate.

Pain Timing Map is a manual decision-support tool for classifying signals before action.
